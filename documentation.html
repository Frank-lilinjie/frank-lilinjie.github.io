    • + This label is not included in any evaluation and treated as void (or in the case of license plate as the vehicle mounted on).

    Data Format

    The annotation file keeps consistent with the COCO format and contains three keys: "images", "categories" and "annotations". 

    "leftImage8bit": {
        "city_name": <str>        -- File name.
        "id": <int>               -- Unique image id.
        "height": <int>         -- Height of the image.
        "width": <int>          -- Width of the image.
}

    "disparity": {
        "city_name": <str>        -- File name.
        "id": <int>               -- Unique image id.
        "height": <int>         -- Height of the image.
        "width": <int>          -- Width of the image.
}

    "gtFine": {
        "name": <str>             -- Unique category name.
        "id": <int>               -- Unique category id.
        "height": <int>         -- Height of the image.
        "width": <int>          -- Width of the image.
        "label":<str>           -- Label of object.
        "polygon":<int[[]]>          -- form of object in image.
    }

    Coarse annotations

    In addition to the fine annotations, we provide coarser polygonal annotations for a set of 20000 images in collaboration with Pallas Ludens. Again, overlayed colors encode the semantic classes. Note that we do not aim to annotated single instances, however, we marked polygons covering individual objects as such.
